## 🌐 پیشنهاد مهم برای اینترنت کاربر

برای گرفتن نتیجه دقیق‌تر، بهتر است برنامه را با **اینترنت اصلی خودتان** اجرا کنید.

استفاده از VPN، Proxy، فایروال سختگیر، آنتی‌ویروس دارای Web Shield یا DNS غیرعادی می‌تواند نتیجه تست‌ها را تغییر دهد. اگر نتایج عجیب بود:

- یک بار با اینترنت مستقیم تست کنید
- DNS را تغییر دهید
- VPN/Proxyهای اضافی را موقتاً خاموش کنید
- مقدار Timeout یا Smart Retry را تغییر دهید
- دوباره با همان لیست تست بگیرید

## 📋 توضیح گزینه‌های برنامه

### ⭐ Option 4 — Config / Manual Specs

اولین گزینه‌ای که بهتر است استفاده شود. در این بخش می‌توانید:

- کانفیگ کامل وارد کنید
- مشخصات را دستی وارد کنید
- نوع شبکه، امنیت، Host، SNI، Path و Port را مشخص کنید
- پیشنهاد اسکنر مناسب را دریافت کنید

اگر کانفیگ ندارید، ورود دستی مشخصات هم پشتیبانی می‌شود.

---

### Option 1 — CDN / WS / HTTP Scanner

برای کانفیگ‌هایی مناسب است که از CDN، Host Header، WS، HTTP، XHTTP یا Front Domain استفاده می‌کنند.

مناسب برای نمونه‌هایی مثل:

- `network=ws`
- `network=http`
- `network=xhttp`
- `security=none`
- کانفیگ‌هایی که Address و Host جدا دارند

این بخش حالت‌های A/B/C را بررسی می‌کند و برای کانفیگ‌های CDN محور مهم‌ترین بخش برنامه است.

---

### Option 2 — Simple SNI Scanner

برای تست ساده SNI در کانفیگ‌های TLS استفاده می‌شود.

مناسب برای:

- کانفیگ‌های TLS مستقیم
- بررسی سریع SNIها
- تست ساده handshake

این گزینه برای CDN/WS پیچیده بهترین انتخاب نیست. اگر کانفیگ شما Host Header یا WS/HTTP دارد، معمولاً Option 1 دقیق‌تر است.

---

### Option 3 — Batch / Multi Config Scan

برای بررسی چند کانفیگ یا چند ورودی استفاده می‌شود. اگر چند تست پشت سر هم دارید، این بخش کمک می‌کند سریع‌تر بررسی کنید.

## 🧠 معنی Mode A / B / C

| Mode | معنی | میزان اعتماد |
|---|---|---|
| A | تست روی IP یا مقصد کانفیگ | کم / فقط کمکی |
| B | تست روی IP واقعی دامنه از DNS خودش | مهم و قابل اعتمادتر |
| C | تست روی IPهای جایگزین/CDN | مهم و قابل استفاده |

### نکته مهم

نتیجه‌ای که فقط **A = OK** باشد، معمولاً قابل اعتماد نیست. برنامه تلاش می‌کند A-only را به عنوان نتیجه معتبر ذخیره نکند. برای ذخیره نتایج واقعی، معمولاً Mode B و C مهم‌تر هستند.

---

## 🧪 Real Connection Test

Real Connection Test فقط به OK شدن ظاهری بسنده نمی‌کند. بسته به نوع کانفیگ، برنامه تلاش می‌کند اتصال واقعی‌تری را بررسی کند:

- برای `ws + none`: بررسی WebSocket/HTTP با Host و Path
- برای `ws/http/xhttp + tls`: بررسی TLS و Host/Path
- برای `tcp + tls`: بررسی TLS handshake
- برای `tcp + none`: بررسی اتصال TCP پایه

---

## 🔁 Smart Retry

Smart Retry برای افزایش دقت تست استفاده می‌شود، اما ممکن است زمان اسکن را بیشتر کند.

| مقدار | معنی |
|---|---|
| 0 | خاموش / سریع‌ترین حالت |
| 1 | معمولی |
| 2 | دقیق‌تر |
| 3 | حداکثری / کندتر ولی پایدارتر |

اگر سرعت برای شما مهم است، مقدار `0` یا `1` پیشنهاد می‌شود. اگر دقت بیشتر می‌خواهید، مقدار `2` یا `3` مناسب‌تر است.

---

## 📦 Cache

Cache باعث می‌شود تست‌های تکراری سریع‌تر انجام شوند. اگر دامنه‌ای قبلاً تست شده باشد، برنامه می‌تواند از نتیجه ذخیره‌شده استفاده کند.

مزیت‌ها:

- سرعت بیشتر در اسکن‌های بعدی
- کاهش تست‌های تکراری
- مناسب برای لیست‌های بزرگ

---

## 🧪 Stability Test

Stability Test بررسی می‌کند که یک نتیجه فقط شانسی OK نشده باشد. یعنی تست می‌تواند چند بار انجام شود تا پایداری نتیجه مشخص شود.

این قابلیت برای تشخیص نتایج ناپایدار مفید است، اما ممکن است زمان اسکن را افزایش دهد.

---

## 📁 فایل‌های لیست SNI

فایل‌های لیست داخل مسیر زیر قرار دارند:

```text
data/sni_lists/
```

نمونه فایل‌ها:

| فایل | کاربرد |
|---|---|
| `default.txt` | لیست عمومی |
| `default_sni.txt` | لیست مخصوص SNI/TLS |
| `iran.txt` | دامنه‌های پیشنهادی ایرانی |
| `custom.txt` | لیست دلخواه کاربر |

وقتی در برنامه گزینه `File` را انتخاب کنید، می‌توانید یکی از این فایل‌ها را انتخاب کنید.

---

## 💾 ذخیره نتایج

نتایج داخل مسیر زیر ذخیره می‌شوند:

```text
data/results/
```

در زمان ذخیره، کاربر می‌تواند انتخاب کند:

- فقط Mode B
- فقط Mode C
- Mode B + C

برنامه نباید نتایج خراب یا A-only را به عنوان نتیجه معتبر ذخیره کند.
